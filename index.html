COMPUTER VISION · REALTIME FEATURED

Realtime Temple Visitor Counter

A deployment-ready AI system that counts temple visitors in real time — without storing any personal data. Built on YOLOv8n for person detection and a custom centroid-based multi-object tracker written from scratch (no external tracking library), the system assigns unique IDs to each individual, tracks them frame-to-frame using pairwise Euclidean distance matrices, and counts crossings over a configurable virtual line to prevent double-counting. A multithreaded Flask backend streams live annotated MJPEG video to a browser dashboard, exposes a full REST API, and auto-saves hourly CSV snapshots — all protected by mutex locks to prevent race conditions between the inference thread and HTTP server. Auto CUDA detection enables FP16 inference on GPU for faster throughput. Privacy by design: no face recognition, no image storage, no biometric data at any stage.

DEPLOYED · LIVE FEATURED

Medical Insurance Cost Predictor

Healthcare costs vary significantly across individuals based on age, BMI, smoking habits, and region. This project builds a Random Forest regression model on 1,338 records to predict insurance charges, achieving R² = 0.84 on test data. The pipeline covers full exploratory data analysis, IQR-based outlier handling (selectively applied — binary/categorical columns excluded), label encoding, and one-hot encoding for region. Smoking status emerged as the dominant predictor, far outweighing other features. The model shows a train/test R² gap indicating mild overfitting on this small dataset — a known limitation of Random Forest without tuning. Hyperparameter optimization via cross-validation is identified as the key next step. The trained model is serialized and served via a clean Streamlit web app — users enter age, gender, BMI, number of children, smoking status, and region to instantly receive a predicted cost in INR.

ML · GEOSCIENCE

Pore Pressure Prediction — Krishna-Godavari Basin

Accurate pore pressure estimation is critical for drilling safety, wellbore stability, and reservoir geomechanics — yet traditional methods rely on expensive physical instruments. This project builds a data-driven alternative using well-log sensor readings from 11,494 depth samples (6m–336m) in India's KG Basin. Standard IQR outlier removal was deliberately avoided — it wiped entire features due to tight well-log distributions — and domain-specific physical thresholds were applied instead, demonstrating real geological domain reasoning. Multiple models were benchmarked: Linear Regression (R²=0.67) and SVR (R²=0.66) confirmed strong non-linearities, while Random Forest achieved R²=0.95. After RandomizedSearchCV tuning with 5-fold cross-validation, both Random Forest and XGBoost stabilised at R²≈0.93 — confirming generalization, not memorization. Feature importance scores were physically interpretable: Stress (54%) and Resistivity (25%) dominated, consistent with geomechanical theory.
